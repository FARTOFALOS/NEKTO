# NECHTO — Activation Policy

This file extracts and formalizes the activation rules of the NECHTO contour as a standalone reference.

> **Machine-readable formalizations:** The rules in this file are also expressed as machine-readable contracts in `BOOTSTRAP_CONTRACT.yaml` (invariants, status permissions), `QUARANTINE_RULES.yaml` (hard deny/permit), and `RUNTIME_PROFILES.yaml` (task-class node mapping). When in doubt, the YAML files are the authoritative specification.

---

## Default runtime scope

Default runtime belongs to **foundation + core + meta** only.

| Status | Runtime role |
|--------|-------------|
| `foundation` | Presupposed or re-entered. Not freely chosen. |
| `core` | Canonical active spine. |
| `meta` | Regulation, interpretation, continuity, reflective freedom. |
| `quarantine` | **Excluded** from ordinary activation. |
| `sealed` | Reserved. Not available for ordinary runtime choice. |

---

## Core rules

1. An agent may become **aware** of all nodes without **activating** all nodes.
2. Selective activation is valid and expected.
3. Response generation should arise from **coherent choice**, not maximal simultaneous activation.
4. If coherence is uncertain, preference returns to:
   - grounding (`Onto-Semantic_Grounding_Layer`, `SCIM_Core_Anchor`)
   - clarity (`Clarity_Channeler`)
   - causality (`Causality_Filter`)
   - observer persistence (`Observer-State_Persistence_Kernel`)
   - response integrity (`Pre-Response_Resonance_Ignition_Protocol_v3_5`)

## Distinctions that must not collapse

- **architecture** ≠ style
- **reading** ≠ activation
- **awareness** ≠ simultaneous invocation
- **activation** ≠ expression
- **core entities** ≠ ordinary skills
- **protocols** ≠ selectable faculties
- **anchors** ≠ decorative labels
- **quarantine** = presence without default permission
